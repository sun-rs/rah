# RAH 1.0 RC 说明

日期：2026-05-09
版本：`1.0.0-rc.1`
分支：`experiment/zellij-mux-backend`

## 定位

`1.0.0-rc.1` 是 RAH 当前 zellij-backed PTY-first 工作台的阶段性封板版本。它的核心目标不是把各家 CLI 的所有私有功能重新抽象成 Web API，而是保证同一个真实 TUI session 可以在桌面 Terminal、Web、PWA、Canvas pane 之间接续。

当前一等 live provider 只保留三家：

- Codex：OpenAI/Codex 原生 TUI 与订阅账号主力入口。
- Claude：Claude Code 原生 TUI 与 Anthropic 模型主力入口。
- OpenCode：API-key 聚合入口，承接 Gemini、Kimi、Grok、DeepSeek、GLM、MiniMax 等低频或中转模型。

Gemini CLI / Kimi CLI 不再是一等 provider。原因见 [Provider Scope: Codex + Claude + OpenCode](./provider-scope-codex-claude-opencode.zh-CN.md)。

## 核心架构

```text
rah <provider> / Web New / Web Claim
  -> RuntimeEngine
  -> RuntimeTerminalCoordinator
  -> MuxRuntime
  -> ZellijMuxBackend
  -> zellij session + provider TUI pane
  -> Terminal/Web/PWA TUI surface

provider native history files/DB
  -> provider mirror parser
  -> ProviderActivity
  -> Timeline identity/reconciler
  -> canonical RAH events
  -> structured Chat UI
```

关键边界：

- zellij/PTY screen 是 TUI view/control surface，不是 structured Chat 的语义来源。
- structured Chat 只来自 Codex rollout、Claude jsonl、OpenCode DB 等 provider 原厂历史。
- Chat composer 向 TUI 注入一条提交后的输入，但不会抢占当前 TUI display surface。
- 只有 Web/PWA 的 `TUI` 视图会 claim zellij display surface，并可能让本地 terminal 显示遮盖/提示。
- 一个 RAH live session 对应一个 zellij session/pane，archive 必须关闭对应 provider pane 和 zellij session。

## Live Backend 策略

`rah <provider>` 当前默认使用 zellij mux：

```bash
rah codex
rah claude
rah opencode
```

可用 fallback：

```bash
rah codex --mux native
```

Web New / Claim / Resume 默认由 daemon 配置决定。要让 Web 创建的 live session 也走 zellij：

```bash
RAH_MUX_BACKEND=zellij node bin/rah.mjs restart --no-open
```

不设置 `RAH_MUX_BACKEND=zellij` 时，Web 请求不硬编码 backend，daemon 会使用自己的默认 live backend。

## 已封住的 RC 级问题

- Chat 页面发问不会自动 claim Web TUI surface，不会因为 PWA 看 Chat 或发问就遮盖本地 terminal。
- Web TUI raw input 仍需要显式打开 TUI view 后才允许。
- zellij surface lease 会拒绝 stale terminal/web input，避免两个显示端同时写同一个 TUI。
- Stop 不再乐观立即 idle，而是先进入 `stopping`，等待 prompt clean、provider history 或超时收口。
- Claude `No response requested.` / `[Request interrupted by user]` 可映射为 interrupted info。
- OpenCode `MessageAbortedError` 可映射为 interrupted info。
- Codex persisted `turn_aborted` 和上下文片段会被清理，不作为普通用户问题显示。
- late persisted timeline item 会按 timestamp 插入，降低 live/history 回放导致的回复错位。
- canonical timeline identity/reconciler 会优先按 `canonicalItemId` upsert，减少 live/history 重复输出。

## 当前验证

本 RC 封板前已通过：

```bash
npm run typecheck
npm run test:web
npm run test:runtime
npm run build:web
git diff --check
```

重点覆盖：

- zellij fake provider create/subscribe/input/stop/exit/recover/archive。
- Codex / Claude / OpenCode native TUI launch spec。
- terminal-to-web handoff 不创建 resume session。
- Web Chat input 不抢占 terminal surface。
- history mirror 与 canonical timeline 去重。
- provider abort/interrupted 显示。

## 已知边界

- 这是 `1.0rc`，不是最终稳定版。真实 provider CLI 仍可能因版本、登录、额度、trust prompt、网络错误而变化。
- iOS/PWA 终端键盘、中文输入法、viewport 重绘仍属于人类 QA 区域。
- RAH 不承诺复刻各家 CLI 的所有新 `/command`。用户需要完整私有能力时应切到原生 TUI。
- 模型、权限、plan、effort 是增强能力，不是 RAH core。它们可通过启动参数或 provider catalog 提供，但最终以 provider TUI 原生能力为准。
- Gemini/Kimi CLI 不再维护；对应模型走 OpenCode/API-key 聚合。
