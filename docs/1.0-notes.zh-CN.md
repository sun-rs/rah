# RAH 1.0 里程碑说明

日期：2026-06-12
版本：`1.0.0`
分支：`main`

## 定位

`1.0.0` 是 RAH 当前 native local-server + TUI mux fallback 工作台的第一个里程碑版本。它的核心目标不是把各家 CLI 的所有私有功能重新抽象成 Web API，而是保证同一个 provider session 可以在桌面 Terminal、Web、PWA、Canvas pane 之间接续。

当前一等 live provider：

- Codex：OpenAI/Codex 原生 TUI 与订阅账号主力入口。
- Claude：Claude Code 原生 TUI 与 Anthropic 模型主力入口。
- Gemini：Gemini CLI 原生 TUI，走 `tui_mux` fallback 与当前 JSON session history mirror。
- OpenCode：API-key 聚合入口，承接 Kimi、Grok、DeepSeek、GLM、MiniMax 等低频或中转模型。

Kimi CLI 仍不作为一等 provider。Gemini 重新接入的边界见 [Session 入口与权限边界](./session-entry-capability-boundary.zh-CN.md)。

## 核心架构

```text
rah <provider> / Web New / Web Claim
  -> RuntimeEngine
  -> RuntimeTerminalCoordinator
  -> provider runtime descriptor
  -> Codex/OpenCode native local server or Claude tmux/TUI mux fallback
  -> Terminal/Web/PWA Chat and optional TUI surface

provider native history files/DB
  -> provider mirror parser
  -> ProviderActivity
  -> Timeline identity/reconciler
  -> canonical RAH events
  -> structured Chat UI
```

关键边界：

- tmux/PTY screen 是 TUI view/control surface，不是 structured Chat 的语义来源。
- structured Chat 只来自 Codex rollout、Claude jsonl、OpenCode DB 等 provider 原厂历史。
- Codex/OpenCode Chat composer 走 provider server control；Claude fallback Chat composer 才通过 TUI bridge 注入提交后的输入，且不会抢占当前 TUI display surface。
- 只有 Web/PWA 的 `TUI` 视图会 claim tmux display surface，并可能让本地 terminal 显示遮盖/提示。
- Claude/Gemini fallback 的一个 RAH running session 对应一个 tmux session/pane；Stop 必须关闭对应 provider pane 和 tmux session。Codex/OpenCode Stop 关闭 RAH 管理的 provider-server session/client。

## Live Backend 策略

`rah <provider>` 当前按 provider 选择主 runtime：

```bash
rah codex
rah claude
rah opencode
```

- Codex / OpenCode 默认走 native local-server，并使用 provider 原生 attach client 连接本地 Terminal。
- Claude 默认走 tmux/TUI mux fallback。

用户 CLI 不再提供 `--mux` 开关。Provider runtime 由 RAH 固定选择：Codex/OpenCode 使用
native local-server，Claude/Gemini 使用 tmux/TUI fallback。底层 native_tui/tui_mux 仍可作为
daemon 内部诊断和回归测试能力存在，但不再是用户启动入口。

## 已封住的 1.0 级问题

- Chat 页面发问不会自动 claim Web TUI surface，不会因为 PWA 看 Chat 或发问就遮盖本地 terminal。
- Web TUI raw input 仍需要显式打开 TUI view 后才允许。
- tmux surface lease 会拒绝 stale terminal/web input，避免两个显示端同时写同一个 TUI。
- Stop 不再乐观立即 idle，而是先进入 `stopping`，等待 prompt clean、provider history 或超时收口。
- Claude `No response requested.` / `[Request interrupted by user]` 可映射为 interrupted info。
- OpenCode `MessageAbortedError` 可映射为 interrupted info。
- Codex persisted `turn_aborted` 和上下文片段会被清理，不作为普通用户问题显示。
- late persisted timeline item 会按 timestamp 插入，降低 live/history 回放导致的回复错位。
- canonical timeline identity/reconciler 会优先按 `canonicalItemId` upsert，减少 live/history 重复输出。
- 长历史首屏和向上翻页走 summary / tail-first paging，普通 Chat UI 不拉取完整 tool/raw payload；工具详情按需 hydrate。
- Codex/OpenCode structured live session 不再靠 history tail polling 驱动实时输出；Claude/Gemini fallback 仍通过 provider transcript mirror 补齐。
- Native local-server 的 client-view TUI replay 不再混入 structured command stdout，避免点击 TUI 时刷出历史命令输出。
- Canvas 采用最多 4 个固定 pane 槽位；布局只改变可见数量，不销毁隐藏 pane 绑定。
- Chat composer 支持粘贴图片为内联缩略图，发送到 provider 时以 data image URL 附加；Chat 历史统一显示图片数量标识，不直接展开 base64。
- 回复里的本地文件链接进入 Inspector file preview，而不是跳转到错误的 HTTP path；host file preview 支持图片、Markdown、CSV/TSV、Notebook 和代码文本。图片 preview 区分访问面：本机/LAN 直接返回原图，远程访问返回 bounded 缩略图，不应因为图片超过普通文本读取上限而显示不可预览。
- Chats 的 Recent/All/Council 使用统一行样式；All catalog 支持增量更新和过滤后删除候选。
- 本机 daemon 默认支持局域网访问；外网稳定访问建议走 Tailscale，细节见远程访问文档。

## 当前验证

本 1.0.0 里程碑发布前已通过：

```bash
npm run test:protocol
npm run typecheck
npm run test:web
npm run test:runtime
npm run build:web
git diff --check
```

重点覆盖：

- tmux fake provider create/subscribe/input/stop/exit/recover/close。
- Codex / Claude / Gemini / OpenCode native TUI / native local-server launch spec。
- terminal-to-web handoff 不创建 resume session。
- Web Chat input 不抢占 terminal surface。
- history mirror 与 canonical timeline 去重。
- provider abort/interrupted 显示。
- long history summary paging、history detail hydration、claim 后 snapshot transfer。
- Canvas fixed pane binding、pane clear/delete/stop 边界。
- PWA/desktop 的 sidebar、inspector、composer、file preview 基本契约。

## 已知边界

- 真实 provider CLI 仍可能因版本、登录、额度、trust prompt、网络错误而变化；RAH 只能在 provider API/TUI 可用的前提下维持接续。
- iOS/PWA 终端键盘、中文输入法、viewport 重绘仍属于人类 QA 区域。
- RAH 不承诺复刻各家 CLI 的所有新 `/command`。用户需要完整私有能力时应切到原生 TUI。
- 模型、权限、plan、effort 是增强能力，不是 RAH core。它们可通过启动参数或 provider catalog 提供，但最终以 provider TUI 原生能力为准。
- Gemini CLI 已恢复为 `tui_mux` provider，结构化 Chat 来自 Gemini JSON session mirror；Kimi CLI 不再维护，对应模型走 OpenCode/API-key 聚合。
- Council 图标、provider 图标、标题栏 pill、状态 pill 属于前端视觉契约；更换图标资源时必须走共享组件，不能在各页面散落裸 SVG。左侧 sidebar 的 Council 图标使用黑色 glyph；其他 Council 图标维持橙色 glyph，并与同位置 provider 图标同规格。
